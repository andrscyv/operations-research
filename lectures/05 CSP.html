Definiciones

Un problema de satisfacción de restricciones (CSP) consiste en:

  • Un conjunto de variables
  • Un dominio para cada variable
  • Un conjunto de restricciones

Los problemas de satisfacción de restricciones se pueden describir en términos de variables y de mundos posibles.

Un mundo posible está descrito por variables (símbolos) las cuales tienen asociado un dominio, el cual es el conjunto de valores que una variable puede tomar.

Si a todas las variables se les asigna un valor de su dominio tenemos un mundo posible: \({X_1 = v_1, X_2 = v_2, \ldots , X_k = v_k}\).

El número de mundos dado \(n\) variables con dominios de tamaño de orden \(d\) es \(d^n\).

Un scope es un conjunto de variables. Una relación en un scope \(S\) es una función de asignaciones en \(S\) a \({true, false}\). Una restricción \(c\) es un scope \(S\) y una relación en \(S\).

Una asignación \(A\) en \(S' \in S\) satisface \(c\) si \(A\) es mapeada a true. En caso contrario \(c\) es violada por la asignación.

Un modelo es un mundo posible que satisface todas las restricciones.

CSP como grafo

  • Los nodos son los valores que puede tomar las variables
  • Los vecino del nodo \(n\) se obtienen seleccionando la variable \(V\) que no está asignada al nodo \(n\) y cuyos vecinos no violan ninguna restricción.
  • El nodo inicial es una asignación vacía de todas las variables
  • El nodo meta es el nodo que asigna un valor a cada variable
    • Nota que este nodo existe si satisface odas las restricciones

Ejemplo:

  • variables: \(A, B, C\)
  • dominio: \({1,2,3,4}\)
  • restricciones: \(A < B\), \(B < C\)

CSP como multigrafo: grafo de restricción de arcos

  • Hay nodos para cada variable
  • Hay otro tipo de nodo para cada restricción
  • Para cada restricción \(c\) y por cada variable \(X\) en el scope de \(c\) hay un arco \((X, c)\)
  • El arco es consistente si para cada valor de la variable se satisface la restricción.

Usando el mismo ejemplo:

  • variables: \(A, B, C\)
  • dominio: \({1,2,3,4}\)
  • restricciones: \(A < B\), \(B < C\)

CSP en python

class CSP:
    def __init__(self, dominios, restricciones):
        self.variables = set(dominios) # quizá sea más eficiente que dominios.keys()
        self.dominios = dominios
        self.restricciones = restricciones
        # Diccionario inverso
        self.variables_restricciones = {variable:set() for variable in self.variables}
        for restriccion in self.restricciones:
            for variable in restriccion.scope:
                self.variables_restricciones[variable].add(restriccion)

    def consistente(self, assigment):
        """assigment es un diccionario de variable: valor """
        return all(c(assigment)
                   for c in self.restricciones
                   if all(variable in assigment for variable  in c.scope))
    
    def __str__(self):
        return str(self.dominios)

    def __repr__(self):
        return f"CSP({self.dominios}, {str([str(c) for c in self.restricciones])})"

from collections import deque

def dict_union(d1, d2):
    """ Simplifica la unión de dos diccionarios """
    d = dict(d1)
    d.update(d2)
    return d

def generar_vecinos(csp, nodo):
    """ nodo es un un diccionario variable: valor """
    variable = list(csp.variables)[len(nodo)]
    vecinos = []
    for value in csp.dominios[variable]:
        assigment = dict_union(nodo, {variable:value})
        if csp.consistente(assigment):
            vecinos.append(assigment)
    return vecinos

def backtracking(csp):
    def goal(csp, nodo):
        return len(nodo) == len(csp.variables)

    mejor_path = []

    start = {}

    frontera = deque()  # En BFS la frontera es una pila

    # Agrego por a derecha, ya que es una cola
    frontera.append((start, [start])) # nodo y path

    while frontera:
        nodo, path = frontera.popleft() # Extraigo por la izquierda
        print(nodo)
        if goal(csp, nodo):
            mejor_path = path
            mejor_solucion = nodo
            yield mejor_solucion, mejor_path
        else:
            vecinos = reversed(generar_vecinos(csp, nodo))
            for vecino in vecinos:
                frontera.appendleft((vecino, path+[vecino]))

alldiff: implementación

  • alldiff es probablemente la restricción global más usada
  • La restricción alldiff(x_1, \dots, x_n) requiere que las variables \(x_1, \ldots, x_n\) tomen distintos valores.
  • alldiff puede implementarse como un problema de matching en un grafo bipartita.
  • El grafo G, tendrá nodos en un lado siendo las variables y en el otro lado los valores de todos los dominos de las variables. El grafo tendrá arcos \((x_j, i)\) si \(i \in D_{x_j}\).
  • Podemos verificar feasibility siel matching de máxima cardinalidad (maximum cardinality matching) cubre a todas las variables \(x_i\). Un algoritmo para encontrarlo es maximun flow (lo veremos más adelante en la clase).
  • Pruning puede ser obtenido con el mismo algoritmo.

Mejorando la búsqueda

  • Usar la información sobre feasibility para realizar el branching
  • Usar el first-fail principle: Intenta primero donde es más probable que falles
    • Variables que tengan un dominio pequeño
    • Variables que estén muy conectadas (la más restringida)
  • Crea árboles de búsqueda más pequeños

# variables

    forall (i in R, j in R i < j)
        reinas[i] != reinas[j]
        reinas[i] + i != reinas[j] + j
        reinas[i] - i != reinas[j] - j

# procedimiento de búsqueda
using {
    forall(r in R) by reinas[r].size(), abs(r-n/2) # Variable/value, first fail: tamaño, restricciones
        tryall(v in R)  # no determinístico si redundancias by col[v].size()
           row[r] = v
}
  • Recuerda, el procedimiento de búsqueda está agregando nuevas restricciones al almacén de restricciones (el cual las valida)
  • En el tryall , si hay un fallo al agregar la restricción, el programa se regresa al anterior tryall y asigna una variable que no se haya usado anteriormente.
  • Si no existe dicho valor, el programa regresa a la última instrucción exitosa no determinística.
  • forall se convierte en una serie de tryall(v in R) row[1] = v
  • tryall se convierte en

try row[r] = 1 or row[r] = 2 or row[r] = 3 etc endtry

Variable/Value labeling

  • labeling se refiere a la asignación
  • Variable/Value indica el orden de elección, i.e.
    • primero seleccionas la variable (la del dominio menor,la más restringida)
    • luego el valor (escoge aquel que deja muchas opciones abiertas)

Domain splitting

  • Idea: Dividir el problema en dos casos disjuntos y resolver cada uno de manera separada. El conjunto de todas las soluciones del problema inicial es la unión de las soluciones de cada caso.
  • Si el dominio de una variable tiene dos elementos, separar es sencillo, si tiene más hay más maneras de dividirlo:
    • Dividir un caso por valor (más rápido)
    • Dividir en dos conjuntos no vacíos, disjuntos (más pruning, no hay que redescubrir cosas)
  • Una manera efectiva es usar AC-3 antes de cada paso de división:
    • Simplifica el problema con AC-3
    • Si el problema no se ha resuelto, selecciona la variable a dividir usando el fast-fail principle.
