Constraint Programming

  • Usar los constraints (restricciones) para reducir el conjunto de valores que cada variable puede tomar.
  • Las restricciones ayudan a reducir el espacio de soluciones.
    • En DP usamos la función objetivo …
  • Indica la estructura del problema de la manera más explícita posible
  • A veces, hay que tomar una decisión cuando no se pueden hacer más deducciones
  • NO es una heurística: con el suficiente tiempo encontrará la solución
  • Se enfoca en feasibility

Definiciones

Un problema de satisfacción de restricciones (CSP) consiste en:

  • Un conjunto de variables
  • Un dominio para cada variable
  • Un conjunto de restricciones

Los problemas de satisfacción de restricciones se pueden describir en términos de variables y de mundos posibles.

Un mundo posible está descrito por variables (símbolos) las cuales tienen asociado un dominio, el cual es el conjunto de valores que una variable puede tomar.

Si a todas las variables se les asigna un valor de su dominio tenemos un mundo posible: \({X_1 = v_1, X_2 = v_2, \ldots , X_k = v_k}\).

El número de mundos dado \(n\) variables con dominios de tamaño de orden \(d\) es \(d^n\).

Un scope es un conjunto de variables. Una relación en un scope \(S\) es una función de asignaciones en \(S\) a \({true, false}\). Una restricción \(c\) es un scope \(S\) y una relación en \(S\).

Una asignación \(A\) en \(S' \in S\) satisface \(c\) si \(A\) es mapeada a true. En caso contrario \(c\) es violada por la asignación.

Un modelo es un mundo posible que satisface todas las restricciones.

CSP como grafo

  • Los nodos son los valores que puede tomar las variables
  • Los vecino del nodo \(n\) se obtienen seleccionando la variable \(V\) que no está asignada al nodo \(n\) y cuyos vecinos no violan ninguna restricción.
  • El nodo inicial es una asignación vacía de todas las variables
  • El nodo meta es el nodo que asigna un valor a cada variable
    • Nota que este nodo existe si satisface odas las restricciones

Ejemplo:

  • variables: \(A, B, C\)
  • dominio: \({1,2,3,4}\)
  • restricciones: \(A < B\), \(B < C\)

CSP como multigrafo: grafo de restricción de arcos

  • Hay nodos para cada variable
  • Hay otro tipo de nodo para cada restricción
  • Para cada restricción \(c\) y por cada variable \(X\) en el scope de \(c\) hay un arco \((X, c)\)
  • El arco es consistente si para cada valor de la variable se satisface la restricción.

Usando el mismo ejemplo:

  • variables: \(A, B, C\)
  • dominio: \({1,2,3,4}\)
  • restricciones: \(A < B\), \(B < C\)

  • B ≠ 3
  • C ≠ 2
  • A ≠ B
  • B ≠ C
  • C < D
  • A = D
  • E < A
  • E < B
  • E < C
  • E < D
  • B ≠ D

from operator import lt, ne, eq, gt

class Restriccion:
    def __init__(self, scope, predicado):
        self.scope = scope
        self.predicado = predicado

    def __call__(self, assigment):
        """ assigment es un diccionario de variable: valor """
        return self.predicado(*tuple(assigment[v] for v in self.scope))

CSP en python

class CSP:
    def __init__(self, dominios, restricciones):
        self.variables = dominios.keys() # quizá sea más eficiente set(dominios)
        self.dominios = dominios
        self.restricciones = restricciones

    def consistente(self, assigment):
        """assigment es un diccionario de variable: valor """
        return all(c(assigment)
                   for c in self.restricciones
                   if all(v in assigment for v  in c.scope))

    def __str__(self):
        return str(self.dominios)

    def __repr__(self):
        return f"CSP({self.dominios}, {str([str(c) for c in self.restricciones])})"

Feasibility testing

Pruning

Dado \(v_i\) en \(D_i\) ¿existe una solución tal que \(x_i = v_i\)?

¿Encontrar valores en los dominios de las variables tales que la restricción se mantienen?

\[ \text{Dado} v_i \in D_i, x_i = v_i \\ \exists v_1 \in D_1, \ldots, v_{i-1} \in D_{i-1}, \\ v_{i+1} \in D_{i+1}, \ldots, v_n \in D_n: \\ c(x_1=v_1, \ldots, x_n=v_n) = true \]
